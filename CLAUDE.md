## Prosjektbeskrivelse

Naturportrett er en webapplikasjon for Oslo kommune som hjelper arkitekter, eiendomsutviklere og saksbehandlere i Plan- og bygningsetaten med ГҘ identifisere biologisk mangfold innenfor influensomrГҘdet (default 500 m, konfigurerbart 100 m вҖ“ 2 km) til en eiendom under utvikling.

For en fullstendig funksjonsoversikt, se [FUNKSJONER.md](FUNKSJONER.md). For endringshistorikk, se [DEVLOG.md](DEVLOG.md).

## Teknisk stack

- **Frontend**: React 18 + Vite (port 5173)
- **Backend**: Express.js API-proxy (port 3001)
- **Designsystem**: Oslo kommune Punkt CSS + Oslo Sans (selvhostet WOFF2)
- **PrimГҰr artsdata**: GBIF (`occurrence/search`)
- **SekundГҰr artsdata**: iNaturalist (research-grade observasjoner, foto, norske navn)
- **AdressesГёk**: Kartverket API (`adresser/v1/sok`, `fuzzy=true`). Default filtrert til Oslo, kan utvides til hele Norge via bryter.
- **Kart**: Leaflet + OpenStreetMap
- **AI**: Anthropic Claude `claude-sonnet-4-6` med fallback-kjede til `claude-opus-4-8` вҶ’ `claude-haiku-4-5` (via server-proxy, aldri direkte fra frontend)
- **E-post-routing**: Resend (for tilbakemeldinger)
- **Deploy**: Railway med persistent volum pГҘ `/data`
- **i18n**: Egen `useT()`-hook + `<SprakProvider>` (NO/EN)

## AdressesГёk

- Default filtrert til Oslo (`kommunenummer=0301`). Bryteren В«Hele NorgeВ» fjerner filteret.
- Alltid `fuzzy=true` pГҘ Kartverket-API-kallet
- `norwegianText.js` normaliserer aeвҶ’ГҰ, oeвҶ’Гё, aaвҶ’ГҘ FГҳR API-kallet
- Fuse.js re-rangerer resultater pГҘ klientsiden
- Influenssone er konfigurerbar via slider (100 m вҖ“ 2 km, default 500 m)

## AI-kall

- Aldri kall Anthropic API direkte fra frontend
- Bruk alltid server-proxyen:
  - `POST /api/claude` for SSE-streamet vurdering (eldre flyt)
  - `POST /api/claude/portrait` for JSON-respons (alle portretttyper)
- System-prompts er i `server/prompts/`:
  - `shared.js` har felles felter + det overordnede `ANTI_HALLUSINERING`-prinsippet, `SYMBIOSE_FIELD`, `TILTAK_FIELD` osv.
  - `naturportrett.js`, `artsportrett.js`, `planteportrett.js`, `naturtypeportrett.js` for hver portretttype
- Server velger system-prompt-tilleggsinstruks basert pГҘ `payload.lang` (norsk default, engelsk hvis brukeren har valgt det)
- Modell-fallback: ved 404 (modell deprecated) bytter `createWithRetry` automatisk til neste i `MODEL_CHAIN`

### Anti-hallusinering вҖ” overordnet prinsipp

KI-troverdighet er den viktigste verdien tjenesten har. Brukerne er fagfolk som vil oppdage spekulasjoner. For alle felter med fabulasjons-risiko (symbioser, lokal forekomst, evidensgrunnlag): **utelat heller enn ГҘ gjette**. Tom array er gyldig svar.

NГҘr UI viser tom symbioser-liste, skal det vises eksplisitt at KI vurderte feltet og ikke fant grunnlag вҖ” ikke at vi skjuler seksjonen.

## Kodekonvensjoner

- Funksjonelle React-komponenter og hooks (ingen klassekomponenter)
- All bruker-tekst gГҘr via `useT()`-hook (NO + EN). Aldri hardkod synlige strenger.
- Engelske variabel- og funksjonsnavn i kode. KI-feltnavn i JSON er norske (`folkenavn`, `vitenskapelig`, `eiendomsKontekst` osv.) вҖ” disse mappes til oversettelser i UI.
- Lov-sitater forblir pГҘ norsk selv nГҘr UI er engelsk (Lovdata er norsk kilde)

- CSS custom properties fra `--oslo-*` paletten, ikke hardkodede farger
- Datakvalitet-tersklene (0.65 / 0.35) er Г©n sannhetskilde i `speciesAggregator.js` og brukes konsekvent i UI
